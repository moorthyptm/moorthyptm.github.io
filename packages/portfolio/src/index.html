01 / About

Engineer who thinks like a product owner.

Technical Lead at TrusTrace, owning frontend architecture across the supply-chain traceability platform — fashion, food, and circular-fashion verticals. Took the codebase through full modularisation into a micro-frontend topology so squads ship independently, drove a cross-product design system that outlasts refactors, and bootstrapped most of the platform's newer modules — DPP, supplier and supply-chain management, audit, Power BI analytics — handing each off to teams while staying on architecture calls. Currently building out AI enablement across the org: RAG-grounded agents, automated human-in-the-loop PR reviews, abstraction layers that keep agent behaviour consistent across tools, and the eval surface to measure it.

The product-owner instinct: I worry less about the elegance of an abstraction than whether the team using it ships faster six months from now. Same for AI tooling — leverage only counts if it survives a quarter of real production use.

05

Playwright Regression Framework

Manual regression: days → hours.

The regression suite was eating multiple days every release. Architected and seeded the team's Playwright framework — page-object pattern, scaffolded for QAs to extend without engineering handholding. Automated one module to 100% coverage and handed it off to the QA team, with multi-actor parallel scenarios (supplier · brand · auditor) running on the same flow. Built on web-first assertions and stable locators — no waitForTimeout antipattern, no flaky tests. As an enabler the framework now spreads across modules, team-owned.
